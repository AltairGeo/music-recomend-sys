# REST API для музыкальных рекомендаций

Сервис хранения, поиска и рекомендаций музыкальных треков на основе акустического анализа.
Принимает аудиофайлы, извлекает метаданные и низкоуровневые признаки, строит векторный индекс для поиска похожих композиций, предоставляет текстовый поиск по метаданным с использованием pg_trgm.

Проект реализован в духе Clean Architecture — доменная логика отделена от инфраструктуры и интерфейсов.

## Основные возможности

  - Управление каталогом треков (метаданные, аудиофайлы)
  - Текстовый нечёткий поиск по названию, исполнителю, альбому, жанру
  - Случайная выдача треков
  - Автоматическое извлечение аудиопризнаков (MFCC, спектральные характеристики, хрома, тоннец и др.)
  - Рекомендации на основе векторного поиска Annoy
  - Загрузка пользовательского файла для мгновенного поиска похожих треков
  - Многопроцессорный конвейер пакетного импорта коллекций
  
  
## Архитектурная схема

```
├── core/                  # Доменные модели и логика
│   ├── audio_processing/  # Извлечение и агрегация признаков
│   ├── recommendation/    # Сервис рекомендаций (порт)
│   └── tracks/            # Модель Track, порты CRUD и хранилища
├── driven/                # Адаптеры к внешним системам
│   ├── database/          # SQLAlchemy модели, DAO, сессия
│   ├── filesystem/        # Локальное файловое хранилище
│   └── recommendation/    # Annoy‑адаптер
├── driver/                # Драйверы
│   └── rest/              # FastAPI, роутеры, DTO, зависимости
├── ingest/                # Конвейер пакетного импорта
├── generics/              # Обобщённые типы (ReadDTO)
└── utils/                 # Вспомогательные модули
```

## Технологии
  - Python 3.14+, FastAPI, Uvicorn
  - PostgreSQL 15+ с расширением pg_trgm
  - SQLAlchemy (async), asyncpg
  - Annoy для приближённого поиска ближайших соседей
  - librosa (акустические признаки), mutagen (метаданные)
  - aiofiles для асинхронной работы с файлами
  - Многопроцессность (встроенный модуль multiprocessing)

# DEV раздел

Локальный запуск проекта:

```bash
# Создание виртуального окружения
python -m venv .venv
source .venv/bin/activate  # или .venv\Scripts\activate на Windows

# Установка зависимостей через uv
pip install uv # Опционально если нету глобального UV
uv sync

# Настройка приложения
cp .env.template .env # Копирование шаблона файла настроек
# Измените данные для подключения к БД и путь хранилища музыки.
```

### CLI
```bash
# Запуск сервера
python main.py serve

# Импорт аудио и его индексация
python main.py ingest ПУТЬ_ДО_МУЗЫКИ --workers 8 # --limit 100
# Индексация автоматически пропускает дубликаты по SHA-256 хэшу файлов.
```
